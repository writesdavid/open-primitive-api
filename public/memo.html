Open Primitive

The federal data layer for AI agents and human decisions

The Market

Ten federal agencies produce the data behind $2T+ in annual US consumer decisions. Transportation, food safety, healthcare quality, pharmaceutical risk, financial regulation. Every American touches this data whether they know it or not.

53% of Americans now use AI assistants for health and safety decisions. That number climbs every quarter. These assistants need ground truth. Without primary federal sources, they guess. They hallucinate. They get people hurt.

AI agents need structured, verified data to function. This is infrastructure, not a feature request.

The RegTech and compliance market sits at $23B in 2026 and projects to $77B by 2034. Thomson Reuters and Wolters Kluwer charge six figures annually for structured access to public data. They built those businesses before agents existed. The agent economy needs a native data layer, and nobody has built it.

The Problem

Federal data is scattered across dozens of agencies. Each publishes in different formats, different schemas, different update frequencies. The FDA uses one API pattern. The EPA uses another. The FAA publishes XML. The BTS ships CSV dumps quarterly. CMS changes its endpoints without notice.

No unified query layer exists. Enigma raised $130M to try. They pivoted away from government data entirely. The problem beat them.

AI models train on summaries of summaries. Fidelity degrades at every layer. When Claude answers "is my water safe?" it has no primary source to reference. It synthesizes fragments from blog posts and news articles. The answer sounds confident. The answer is often wrong.

Every developer who wants federal data rebuilds the same parsing, the same error handling, the same caching. Thousands of engineers solving the same problem independently, each with partial coverage.

The Product

Open Primitive connects 13 federal data domains through one unified REST API. Water quality. Food recalls. Drug adverse events. Hospital safety scores. Vehicle complaints. Airline reliability. Nursing home inspections. Each domain normalized to a consistent schema.

The MCP server exposes 13 tools. AI agents call them directly. No prompt engineering required. No retrieval-augmented generation workarounds. The agent asks for water violation data in a ZIP code and gets structured federal records with source URLs.

The A2A agent card makes Open Primitive discoverable by agent orchestrators. Agents find us without human configuration.

Cross-domain queries set this apart. A composite safety score for any US ZIP code, joining water quality, hospital ratings, food inspection history, and drug event frequency. Nobody else joins federal datasets across agencies.

Every API response includes the domain, source agency, source URL, and freshness timestamp. Provenance is not optional. Historical archiving captures time-series data that federal agencies themselves delete or overwrite. Within 12 months of operation, this archive becomes irreplaceable.

The Traction

The API is live at api.openprimitive.com. Ten federal agencies connected and returning data. Agent traffic detection distinguishes machine callers from human browsers. Webhook alerting pushes new recalls and violations to subscribers the moment federal agencies publish them.

Seven consumer-facing tools run at openprimitive.com. Each one demonstrates a data domain: flight reliability, vehicle defects, water violations, food recalls, drug side effects, hospital quality, supplement efficacy. These tools validate the API, generate organic traffic, and prove the data works at the point of decision.

The entire platform was built and shipped by one engineer. Every scraper, every parser, every API endpoint, every consumer tool. The technical foundation exists. What comes next is scale.

The Moat

Four layers protect this business, and each one compounds over time.

Cross-domain composites. Joining water quality, hospital scores, drug events, and food inspections per ZIP code requires normalizing data from agencies that were never designed to talk to each other. This integration work is tedious, unglamorous, and essential. Competitors would need to rebuild it from scratch.

Historical archive. Federal agencies overwrite current data. They do not maintain accessible time-series. Every day Open Primitive runs, the archive grows. After 12 months, we hold longitudinal federal data that exists nowhere else. After 24 months, the switching cost is permanent.

Agent network effects. MCP and A2A registration puts Open Primitive in agent tool registries. Once agents discover and rely on these tools, they route queries here by default. Each new agent integration increases the value for every other agent in the network.

Query intelligence. Logs reveal what agents actually ask for. Which domains, which geographies, which cross-references. This data shapes product decisions, surfaces unmet demand, and creates a flywheel that no competitor can replicate without equivalent traffic.

The Competition

Enigma raised $130M to structure government data. They abandoned the mission and pivoted to KYB (know-your-business) verification. The government data problem remains unsolved.

Socrata (Tyler Technologies) sells data portals to governments. Their customer is the agency, not the developer. Not the agent.

data.gov is a metadata catalog. It points to datasets. It does not serve queryable data.

OpenFDA covers a single agency. Dataset updates were suspended in March 2025. Single-agency solutions cannot provide cross-domain intelligence.

Nobody builds unified cross-agency APIs designed for agent consumption. The market is open.
