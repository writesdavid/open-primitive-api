Open Primitive Protocol

The data layer of the agent internet

The Market

Bot traffic will exceed human traffic by 2027. Cloudflare's CEO said it plainly on March 19, 2026: the internet is becoming a machine-to-machine network. The browsing era is ending. The agent era has started.

The Big Four will spend $700B on AI infrastructure in 2026. Salesforce Agentforce, ServiceNow agent workflows, Microsoft Copilot Studio. Every major platform now sells agent orchestration. Every agent needs data to act on. Trusted data. Verifiable data. Data with provenance.

RegTech and compliance spending sits at $23B in 2026 and projects to $77B by 2034. Thomson Reuters and Wolters Kluwer charge six figures annually for structured access to public data. They built those businesses before agents existed.

AI-native data companies trade at 25-30x revenue. Traditional SaaS trades at 6x. The market pays a premium for companies that make data agent-consumable. That premium exists because almost nobody does it well.

The Problem

Agents consume data with no provenance, no freshness guarantees, no confidence scores. An agent pulls a food recall from a cached search result. The recall was updated six hours ago. The agent doesn't know. The user doesn't know. Decisions get made on stale data dressed up as fact.

The agent internet has no data envelope standard. MCP defines how agents call tools. A2A defines how agents talk to each other. Nobody defines how data providers make data trustworthy for agent consumption. The plumbing exists. The water quality doesn't.

Federal data is the sharpest version of this problem. The FDA suspended OpenFDA dataset updates in March 2025. The government deleted 3,000+ datasets from federal repositories. Data.gov went offline during the October 2025 shutdown. Agencies publish in incompatible formats, change endpoints without notice, and overwrite historical records. No cross-agency integration exists.

Every developer who touches federal data rebuilds the same parsing, the same error handling, the same caching. Every agent that needs government records improvises. The absence of a standard guarantees fragility.

The Protocol

The Open Primitive Protocol (OPP) defines how data providers make data agent-consumable, verifiable, and discoverable. Three components.

ComponentFunction
ManifestMachine-readable declaration of available domains, update frequencies, schema versions, and data quality grades. Agents read the manifest to know what a provider offers and how fresh it is.
Signed EnvelopeEvery response wrapped in a cryptographic envelope: source agency, retrieval timestamp, confidence score, citation URL, and Ed25519 signature. Agents verify provenance without trusting the provider on faith.
Standardized EndpointConsistent query patterns across all data domains. Natural language routing, structured filters, pagination, and cross-domain joins. One interface regardless of source agency.

OPP is compatible with MCP, A2A, OpenAPI, JSON-LD, and W3C Data Integrity. It does not replace agent communication protocols. It fills the gap they left: data trust.

Block is a founding member of AAIF. OPP has a direct path to standards body proposal. First to define the data envelope standard holds the TCP/IP position of the agent internet.

The Implementation

The reference implementation is live at api.openprimitive.com. 16 federal data domains. 11 agencies. Running in production.

The MCP server exposes 13 tools. AI agents call them directly. No prompt engineering. No RAG workarounds. The agent asks for water violations in a ZIP code and gets structured federal records with source URLs, retrieval timestamps, and data quality grades.

Natural language query routing maps plain questions to the right domain, the right agency, the right endpoint. Cross-domain intelligence joins datasets that were never designed to talk to each other: composite safety scores by ZIP code, location profiles that combine water quality with hospital ratings with food inspection history.

Every response is citation-ready. Source agency, source URL, freshness timestamp, reasoning hints. Agents pass these citations through to users. Trust propagates from the primary source to the final answer.

The Moat

Protocol definition. First to define the data envelope standard sets the terms for every provider that follows. OPP becomes the specification other providers adopt. This is the TCP/IP position, not the application layer.

Historical archive. Federal agencies overwrite current data. They do not maintain accessible time-series. Every day Open Primitive runs, the archive grows. After 12 months, this longitudinal data exists nowhere else. After 24 months, the switching cost is permanent. Historical data compounds daily and cannot be replicated retroactively.

Cryptographic provenance. Ed25519 signed responses create a verifiable chain from federal source to agent consumer. Competing services can serve data. They cannot prove it unchanged from the source without rebuilding the signing infrastructure.

Cross-domain joins. No single agency joins its data with other agencies. Water quality plus hospital ratings plus drug events plus food inspections for a single location requires normalization work across incompatible schemas. Tedious. Essential. Expensive to replicate.

Editorial judgment. Which data benefits from AI translation and which benefits from reading raw? This question has no algorithmic answer. The learning layer at openprimitive.com captures this judgment. It compounds into a taxonomy of data legibility that no pure-API competitor will build.

The Competition

us-gov-open-data-mcp. 300+ tools. Raw wrapper around federal APIs. No protocol layer, no provenance, no confidence scores, no human-facing layer. A firehose with no filter.

GovInfo MCP. Official but single-agency. Congressional and regulatory documents only. No cross-domain intelligence.

Google Data Commons MCP. Statistical aggregates from Census and World Bank. No editorial judgment. No real-time federal data. No signed provenance.

Thomson Reuters / Wolters Kluwer. Six-figure annual contracts for structured public data. Cannot go free. Cannot serve agents at $0.005/call. Business model prevents them from competing on price or protocol openness.

Nobody defines the data envelope standard. Nobody combines protocol specification with reference implementation with human editorial layer. The market is open.

The Business
