OPP Payments

Protocol-level monetization. Payment is part of the spec, not bolted on.

The Model

OPP supports three payment modes. Providers declare their mode in the manifest.

Mode 1

Free

Public domain data. No payment required.

manifest.authentication.type = "none" or "api-key" with free registration.

Used for government data, open datasets, public records. The default for most OPP providers today.

Mode 2

x402 Micropayments

Per-request payment. The agent pays for each call.

manifest.authentication.type = "x402"

manifest.pricing = { currency: "USDC", perRequest: "0.005" }

Flow: Agent sends GET. Server returns 402 with x402 headers. Agent signs payment. Server delivers data.

Settlement: USDC on Base. About 200ms.

Mode 3

Subscription

Enterprise access. API key with paid tiers.

manifest.authentication.type = "api-key" with paid tiers.

manifest.pricing = { tiers: [{ name: "free", limit: 500 }, { name: "pro", price: 99, limit: 100000 }] }

Manifest Pricing Fields

Providers declare pricing in opp.json. Agents read this before making a single request.

{
  "pricing": {
    "model": "free" | "x402" | "subscription" | "hybrid",
    "currency": "USDC",
    "perRequest": "0.005",
    "freeLimit": 500,
    "tiers": [
      { "name": "free", "limit": 500 },
      { "name": "pro", "price": 99, "limit": 100000 }
    ]
  }
}

The model field tells agents what to expect. "hybrid" means the provider offers a free tier and x402 for overages.

x402 Flow

Four HTTP exchanges. No redirects, no OAuth, no token refresh.

1

Agent sends a standard request

GET /v1/drugs?name=aspirin HTTP/1.1
Host: api.openprimitive.com
Accept: application/json
2

Server returns 402 Payment Required

HTTP/1.1 402 Payment Required
X-Payment-Required: true
X-Payment-Amount: 0.005
X-Payment-Currency: USDC
X-Payment-Network: base
X-Payment-Address: 0x1a2b3c...provider-wallet
X-Payment-Scheme: exact

The agent now knows the price, currency, network, and destination. All machine-readable.

3

Agent signs and retries

GET /v1/drugs?name=aspirin HTTP/1.1
Host: api.openprimitive.com
Accept: application/json
X-PAYMENT: eyJwYXlsb2FkIjp7InR... // signed USDC transfer

The X-PAYMENT header contains a signed EIP-712 message authorizing the USDC transfer on Base.

4

Server verifies, settles, delivers

HTTP/1.1 200 OK
Content-Type: application/json

{
  "data": { ... },
  "source": { "agency": "FDA", "url": "https://api.fda.gov/..." },
  "freshness": { "retrieved": "2026-03-19T14:00:00Z" },
  "payment": { "settled": true, "txHash": "0xabc..." }
}

Standard OPP envelope. The payment field confirms settlement.

Why Protocol-Level Payments

When payment is part of the protocol, agents can programmatically discover price, evaluate cost vs value, and pay without human intervention.

An agent planning a research task reads 40 OPP manifests. 12 are free. 28 charge between $0.001 and $0.05 per request. The agent calculates total cost, compares it to its budget, and executes. No signup forms. No credit card entry. No approval workflow.

This is how the agent economy works. Price is metadata. Payment is a header. Data flows.

For Providers

Add pricing to your manifest. If you use x402, install the @x402/server package. OPP handles discovery. x402 handles payment. You handle data.

// server.js — add x402 middleware
const { x402 } = require('@x402/server');

app.use('/v1', x402({
  payTo: '0x1a2b3c...your-wallet',
  amount: '0.005',
  currency: 'USDC',
  network: 'base'
}));

app.get('/v1/your-data', (req, res) => {
  // Payment already verified by middleware
  res.json({ data: fetchYourData() });
});

Three steps. Declare pricing in opp.json. Add the middleware. Ship.
