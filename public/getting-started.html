Get started with OPP

Three paths. Pick the one that fits. Each gets you to a working result in 10 minutes.

Path 1 — Use Data

For agent builders

Query any OPP provider. Get verified data with provenance.

1. Pick an endpoint

Every OPP provider exposes domain-specific endpoints. Hit one.

curl https://api.openprimitive.com/v1/drugs?name=aspirin

2. Read the envelope

Every response wraps data in a standard envelope. Six fields you can trust.

{
  "domain": "drugs",
  "source": "FDA Adverse Event Reporting System",
  "freshness": "2026-03-18T00:00:00Z",
  "confidence": 0.94,
  "citations": [
    "https://api.fda.gov/drug/event.json"
  ],
  "proof": "sha256:9f86d081884c7d659a2feaa0..."
}

domain tells you the data category. source names the federal origin. freshness marks when the provider last pulled from source. confidence scores data completeness. citations link to raw federal endpoints. proof is a content hash you can verify independently.

3. Integrate

MCP

// claude_desktop_config.json
{
  "mcpServers": {
    "opp": {
      "command": "npx",
      "args": ["@opp/mcp-server"],
      "env": {
        "OPP_API_URL": "https://api.openprimitive.com"
      }
    }
  }
}

Python

pip install opp-client

from opp_client import OPP
client = OPP()
result = client.query("drugs", name="aspirin")
print(result.envelope.source)

JavaScript

const res = await fetch(
  "https://api.openprimitive.com/v1/drugs?name=aspirin"
);
const { data, envelope } = await res.json();

// Verify proof before using data
console.log(envelope.source);
console.log(envelope.confidence);

LangChain

from langchain_opp import OPPToolkit

toolkit = OPPToolkit(
    base_url="https://api.openprimitive.com"
)

Path 2 — Provide Data

For data publishers

Make your API agent-readable in 50 lines.

1. Create your manifest

Add /.well-known/opp.json to your domain. This tells agents what you serve.

{
  "opp_version": "0.1",
  "provider": "your-org-name",
  "domains": ["water"],
  "endpoints": [
    {
      "path": "/v1/water/contaminants",
      "method": "GET",
      "description": "Returns contaminant levels for a water system by ID.",
      "params": {
        "system_id": "string, required"
      }
    }
  ],
  "source": "EPA Safe Drinking Water Information System",
  "contact": "data-team@your-org.com"
}

2. Add the envelope to responses

Before: raw data. After: data with provenance. Agents need both.

// BEFORE — bare response
app.get("/v1/water/contaminants", (req, res) => {
  const data = getContaminants(req.query.system_id);
  res.json(data);
});

// AFTER — OPP envelope
app.get("/v1/water/contaminants", (req, res) => {
  const data = getContaminants(req.query.system_id);
  res.json({
    data,
    envelope: {
      domain: "water",
      source: "EPA SDWIS",
      freshness: lastFetchTimestamp,
      confidence: computeConfidence(data),
      citations: ["https://sdwis.epa.gov/..."],
      proof: sha256(JSON.stringify(data))
    }
  });
});

3. Sign responses

Signing lets consumers verify your data was not altered in transit.

import { generateKeyPairSync, sign } from "crypto";

// Generate once, store the private key securely
const { publicKey, privateKey } = generateKeyPairSync("ed25519");

function signEnvelope(envelope) {
  const payload = JSON.stringify(envelope);
  const signature = sign(null, Buffer.from(payload), privateKey);
  return {
    ...envelope,
    signature: signature.toString("base64"),
    publicKey: publicKey.export({ type: "spki", format: "pem" })
  };
}

4. Register

Tell the registry your provider exists.

curl -X POST https://api.openprimitive.com/v1/registry/register \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-domain.com",
    "manifest": "https://your-domain.com/.well-known/opp.json",
    "contact": "data-team@your-org.com"
  }'

5. Validate

Paste your domain into the validator. It checks your manifest, envelope structure, and signature chain.

https://openprimitive.com/validate.html

The validator returns pass/fail for each requirement. Fix what fails. Ship when it passes.

Path 3 — Build on the Protocol

For contributors

Extend OPP. Propose spec changes. Build the ecosystem.

Start here

  • GitHub repository Source code, issues, pull requests
  • Protocol specification Envelope format, discovery, signing, verification
  • SDK directory Client libraries for Python, JavaScript, and Go
  • Contributing guide How to propose changes, submit providers, report bugs

Clone and run locally

git clone git@github.com:writesdavid/open-primitive-api.git
cd open-primitive-api
npm install
npm run dev

The API runs at http://localhost:3000. Every endpoint documented in the spec works locally with bundled test data.

What needs building

Open issues tagged good-first-issue on GitHub. Priority areas: new domain providers, SDK improvements, envelope verification tooling, and test coverage for edge cases in signature validation.
